# Omega-TIB 独立质量审计

## 一、项目理论基石（The OMEGA Manifesto）

> 以下是本项目的核心洞察。它是所有架构决策的出发点。
> **你的任务不是质疑这个洞察——而是发挥你的能力，
> 判断当前实现是否配得上这个洞察，并指出如何更好地实现它。**

在高度博弈的 A 股市场中，大部分时间的行情演化是由散户的无序交易
和宏观随机事件驱动的。此时，市场处于极高的**时间有界熵
（Time-bounded Entropy）**状态，表现为纯粹的噪声与随机游走。

然而，**主力（具有资金、信息和筹码优势的机构）**的行为是高度
目的性、结构性和有预谋的（例如建仓、洗盘、主升浪拉升）。

**压缩即智能（Compression is Intelligence）**
主力的行为轨迹在数学上并非随机噪声，而是高度可压缩的结构化信息，
即 Epiplexity（认知复杂度/结构信息）。
发现主力，本质上就是在有限的算力下，从高熵的市场噪声中剥离出
这些低复杂度的、可复用的规律。

**高维拓扑的降维惩罚（Square-root Law）**
主力的资金调度绝非单点的线性操作，而是跨股票、跨板块、跨时间周期
的立体协同。
如果在数学上将这些多维数据强行降维成一维的时间序列，根据方根法则
（Ω(√N)），原本紧凑的主力行为拓扑将被撕裂，产生巨大的嵌入膨胀
（Embedding Dilation）。

**空间有界熵与有限窗口捕获（Space-bounded Entropy & Finite Window）**
人类与机器的观测能力都受限于有限窗口（Finite Window, W > 1）。
要无损地提取主力的 Epiplexity，必须保留市场数据的高维拓扑结构，
利用**拓扑数据分析（TDA）**在高维流形上直接进行"框选"，从而
避开一维序列带来的空间有界熵急剧增加的惩罚。

一旦在正确的维度和窗口下捕捉到主力的低熵结构，跟随其买卖方向
即可（也可反之，看资金量的多少而定）。

## 四、选择题 — 数据足够做明确判断

以下问题的答案可以直接从底稿数据推断。请给出明确的
**是/否/A或B** 判定 + 引用具体证据。不接受"取决于情况"。

**Q1. hd=64 是否必须在 Phase 13 管线上重新测试？**
hd=64 的唯一实证是 Phase 6 HPO (T29 IC=0.066)，但 INS-072
宣布作废 (C-062 torch.compile bug)，Phase 14 复测 Rank IC=0.0023。
现在 Phase 13 管线可信了（IC Loss + Pre-LN + AttentionPooling）。
→ **是否必须重测 hd=128/256？是 or 否？**

**Q2. batch=256 的 IC 梯度信噪比是否 < 1？**
Pearson IC 采样标准误差 ≈ 1/√N。N=256 → SE≈6.25%。
信号 Rank IC=2.9%。
→ **梯度信号是否被采样噪声淹没？请精确计算。**
→ 增大到 B=1024 和 B=2048 时，SE 分别是多少？

**Q3. SRL 压缩是否充分？**
Phase 14 Step 2: Arm A (仅 SRL, 6 维) Rank IC=+0.0122 vs
Arm B (SRL + 原始宏观, 8 维) +0.0064。
→ **SRL 是否已充分提取 V_D/σ_D 信息？是 or 否？**

**Q4. lambda_s=0 是否正确？**
6 Phase 实证: 每次 lambda_s > 0，信号变差或死亡。
Phase 13 lambda_s=0 首次成功。Kurtosis=2006。
→ **在此数据分布下，L1 惩罚是否反信号？是 or 否？**
→ 如果"是"，给出 L1 在肥尾分布下惩罚尾部信号的数学论证。

**Q5. MLP baseline 是否应在进一步架构优化前执行？**
项目从未跑过简单 MLP baseline（同输入、同数据、无 SRL/拓扑）。
如果 MLP 达到 Rank IC=0.025，则 Omega-TIB 整个物理+拓扑架构
仅贡献 0.004。
→ **是否应立即执行 MLP baseline？是 or 否？**

**Q6. Phase 13 训练曲线 E9→E14 是什么状态？**
E9 (best): Rank IC=+0.0292。E10-E14 在 +0.012~+0.021 震荡。
无 patience-based early stopping。
→ **这是 [A] 过拟合、[B] 噪声主导、还是 [C] 收敛后的方差？**
→ 你的判据是什么？

## 五、开放题 — 值得深入讨论

以下问题没有唯一正确答案。请给出推理过程和最佳猜测方向。
有价值的猜测 > 谨慎的废话。

**Q7. 单层注意力 + 零跨窗口通信 — 这是否是 Rank IC=0.029 的
   硬天花板？**
当前 5 个 (32,10) 窗口完全隔离。如果机构建仓跨越 40-50 bars
（>1 个窗口），模型物理上不可能将其识别为连续模式。
INS-070 已提出但被推迟。
→ 如果实现 Swin-style shifted window（2 层），你预期 IC 提升
   多少？给一个合理范围和推理。

**Q8. IC Loss 在 Kurtosis=2006 下是否最优？**
IC Loss 对每个样本等权处理。项目哲学是 Taleb 式——
"低胜率 + 尾部暴利"。但 Loss 函数在优化时不区分
+500BP 尾部事件和 +2BP 平庸事件。
→ 这两者之间是否存在根本性矛盾？
→ 如果存在，提出一个具体的替代 Loss 设计（代码级精度）。

**Q9. 24.6K 参数 — 是 Karpathy 极简的胜利，还是容量不足？**
项目哲学："代码行数是负债，简单 > 聪明。"
但同类量化 ML 模型通常 100K-10M 参数。
→ 在 OMEGA Manifesto 的框架下（"主力行为是低熵可压缩的"），
   极小模型是哲学正确的。但 Rank IC=0.029 是否暗示
   模型太小以至于丢弃了部分可压缩信号？
→ 压缩的极限在哪里？如何区分"成功过滤噪声"和"过度压缩信号"？

**Q10. SRL 的 δ=0.5 在 A 股是否成立？**
平方根法则的原始实证来自美股/日股机构交易。
A 股特殊: T+1 制度、涨跌停、散户 >50%。
→ 如果 δ 偏离 0.5（例如 0.4 或 0.6），对 q_metaorder 的
   数值范围和分布有什么定性影响？
→ 这是否值得实验验证？成本多少？

**Q11. 正则化全面缺失 — 设计选择还是历史遗忘？**
weight_decay=1e-5 ≈ 0, dropout=0, mask_prob=0.0, lambda_s=0。
lambda_s=0 有 6 Phase 实证支持。
但其他三项从未被讨论过。
→ 对一个在 9.96M 样本上训练 75K 步的 24.6K 模型，
   加入轻量正则化（dropout=0.05, WD=0.01）的预期影响？

**Q12. Rank IC=0.029 在 A 股 L1 tick + T+1 隔夜 的语境下
   处于什么水平？**
→ 与公开文献或你所知的同类研究相比。
→ D9-D0=7.00 BP vs 交易成本 ~25 BP 的差距意味着什么？
→ 需要多少 IC 才能覆盖成本？给出量化关系。
